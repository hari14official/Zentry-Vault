TripleDES Process (EDE)

Stage 1 ($K1$): Encryption using the first 8-character sub-key.
Stage 2 ($K2$): Decryption using the second sub-key (further scrambles the message).
Stage 3 ($K3$): Final Encryption using the third sub-key.
Result: Secure 64-bit encrypted package.

TripleDES Decryption Process (DED)

Inverse of encryption: Decrypt ($K3$) → Encrypt ($K2$) → Decrypt ($K1$). Each sub-key must be exactly 8 characters.
